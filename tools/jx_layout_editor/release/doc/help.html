Overview of JX Layout Editor

JX Layout Editor provides a graphical editor for layout out widgets in a window. It replaces fdesign and jxlayout, and it can import your existing layout files.

The Layout window

When you create a new layout, you start with an empty canvas.

Creating and placing new widgets is done by meta-dragging. After you release the mouse button, you choose what kind of widget you want, and then you get the appropriate configuration dialog.

All widgets need a variable name and sizing configuration. You can specify whether the variable is a member of the class, a pre-declared variable, or a plain local variable. The configured sizing is indicated by marking fixed edges orange in the layout window. Additional configuration depends on the type of widget.

Select a widget by clicking it. If a single widget is selected, it will display drag handles for resizing.

You can move widgets by selecting and then dragging them. In the same window, hold down Meta to place a copy of the widget instead of moving. In a different window, hold down Meta to move the widget instead of copying it.

You can also move and resize widgets with the arrow keys. Plain arrow keys shift the selected widgets. Meta-arrow keys shift the right and bottom edges. This enables you to adjust the sizes of multiple widgets in one operation. You can resize the window using the menu items on the Layout menu.

You can turn of the default snap-to-grid behavior for any operation by holding down the Shift key.

Some widgets are containers, e.g., JXWidgetSet, JXScrollbarSet, and JXToolBar. You can place widgets inside these containers just like on the main canvas. To move the container widget itself, hold down the Control key while clicking on a blank part of its canvas. Control-double-clicking opens the configuration dialog for the container widget.

Note: Selected widgets must all have same parent. You cannot add a widget to the current selection if it has a different parent.

For widgets that support keyboard shortcuts, you can specify multiple shortcuts by concatenating them without any separator, e.g., ^C#.

Tab order for input fields is indicated by small number tags. You can adjust the tab order using the Increment/Decrement tab index items on the Arrange menu.

When you save a layout, the matching source and header files will automatically be updated. Any strings will be written to a file in the strings directory of the project.

Note: Code tags must be valid C++ identifiers because they are using in string keys.
