Overview of NPS Subversion® Client

This programs simplifies the most common Subversion tasks:

  • Retrieving updates
  • Managing and committing changes

  • Analyzing change history
  • Comparing revisions

  • Browsing the central repository

Main Window

A window can be opened for either a remote repository or a local working copy. In the former case, the window will start one tab, "Repo", which shows the contents of the repository. In the latter case, the window will start two tabs, "Repo" and "Status". The second tab shows the output from svn status.

Use the File menu to open a new window with either a remote repository or a local working copy.

Repo Tab

This tab displays the contents of the remote repository.

Click on an item to select it. Ctrl-click or Shift-click to select additional items.

To rename an existing item, click on the name. The Actions menu allows creations of new directories and removal or duplication of selected items.

If the window is linked to a local working copy, you can find the corresponding file on disk by selecting an item and then invoking "Show selected files in file manager" on the File menu.

The Info menu lets you view the commit history of an item, the properties set on an item, and the differences between the current and previous contents of the item.

If you select an item and then invoke "Copy full path" on the Edit menu, the full repository URL of the selected item will be placed on the clipboard.

Right-clicking on an item gives a menu of the most common actions.

Use "Refresh display" on the Actions menu to update the list after somebody else commits to the repository. The list is automatically updated when you commit from the Status tab.

Status Tab

This tab displays all the uncommitted changes in the local working copy. New files are displayed in blue. Deleted files are displayed with strike-through. Unmanaged files are displayed in gray. Conflicts and missing files are displayed in red.

Click on an item to select it. Ctrl-click or Shift-click to select additional items.

The Actions menu lets you perform Subversion actions on the selected items. The Info menu lets you analyze commit history, view properties, and compare with previous revisions. (Code Crusader provides a more general interface to svn diff.)

You can find the corresponding file on disk by selecting an item and then invoking "Show selected files in file manager" on the File menu.

If you select an item and then invoke "Copy full path" on the Edit menu, the full path to the selected item will be placed on the clipboard. Invoking "Copy" on the Edit menu places the relative path to the item on the clipboard.

Right-clicking on an item gives a menu of the most common actions. Note that the "Ignore..." item places the name of the selected item on the clipboard, so you can paste it into the Subversion properties editor.

Use "Refresh display" on the Actions menu to update the list after you make changes to your working copy.

Commit Tab

A new Commit tab is created every time you commit changes from the Status tab. The Commit tab shows the output of svn commit. You can close the tab by clicking the red [x] that appears when you move the mouse over the tab.

Log/Info Tab

A new Log/Info tab is created every time you invoke "Info & Log" on the Info menu. This shows the output of svn info and svn log. You can close the tab by clicking the red [x] that appears when you move the mouse over the tab.

If you right-click on the revision number in the upper left corner of a log entry, you can quickly compare the selected revision with other, common revisions and also get details on what was committed.

The right-click menu also provides a shortcut to browse the repository as it appeared in that revision.

You can copy text from this tab by selecting it and then invoking "Copy" on the Edit menu.

Use "Refresh display" on the Actions menu to update the list after you or somebody else commits to the repository.

Properties Tab

A new Properties tab is created every time you invoke "Properties" on the Info menu. This shows the output of svn proplist. You can close the tab by clicking the red [x] that appears when you move the mouse over the tab.

Special properties, e.g., svn:ignore, are displayed in bold.

Double click a property to edit it. Create a new property by using the "New property..." item on the Actions menu. Remove an existing property by selecting it and then invoking "Remove selected properties..." on the Actions menu.

Clean Up Tab

A new Clean Up tab is created if a commit fails. This displays the output (usually nothing) from svn cleanup. You can close the tab by clicking the red [x] that appears when you move the mouse over the tab.

Preferences

The Integration dialog lets you configure:

  • The command to use for viewing the output of svn diff
  • The command to execute after Subversion has modified files

The default is to use Code Crusader, which provides a graphical view of the difference between revisions. If you choose the "Command Line" option, diffs will be displayed by opening a terminal. If you use a different IDE, select "Custom" and enter the commands required to interface with your IDE.

The Configure Toolbar dialog lets you choose which menu items to display as buttons below the menu bar.

The File Bindings dialog allows you to configure which program to use for viewing different types of files. Click the Help button in the dialog for more information.

The Web Browser Preferences dialog is primarily useful for specifying the command to use to open a file manager window. The default is systemg. On OS X, this script will display the file in the Finder.

Macintosh/Windows/X preferences provides some miscellaneous preferences for adjusting the program's behavior.

"Save window setup as default" stores the position and size of the window, so new windows will open in the same location and have the same size.
